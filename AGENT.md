# 项目背景介绍

## 项目概览

这是一个微信原生小程序词汇学习项目，当前定位是“本地词书 + 本地学习进度 + 学习/复习闭环”的背单词应用。项目主线已经成型，围绕考研、四六级、产品/技术英语等词书，提供今日学习、到期复习、错词回看、收藏巩固和词书切换等能力。

当前仓库里的 `README.md` 仍然是脚手架默认说明，不能真实反映业务背景；后续协作应以 `miniprogram/` 里的实际实现为准。

## 代码位置

- 实际小程序源码在 `miniprogram/`
- 页面入口在 `miniprogram/pages/`
- 全局状态和学习逻辑在 `miniprogram/utils/state.ts`
- 数据模型在 `miniprogram/utils/types.ts`
- 词书种子数据在 `miniprogram/data/`
- 全局配置在 `miniprogram/app.json`

注意：当前工作区外层目录只有少量配置文件，真正的项目根目录是本文件所在的 `xiaochengxu-cibei-main/`。

## 技术栈与运行方式

- 技术形态：微信原生小程序
- 语言：TypeScript + 已存在的同名 JavaScript 产物
- 工具链：微信开发者工具，`package.json` 里只保留了轻量 `typescript` 和 `miniprogram-api-typings`
- 数据存储：本地 `wx.getStorageSync` / `wx.setStorageSync`
- 音频能力：学习页通过有道发音接口播放单词音频

项目不是云开发架构，也没有后端接口接入痕迹；当前版本主要是单机、本地状态驱动。

## 当前产品结构

`app.json` 定义了 5 个页面，其中 4 个在 tabBar 中：

- `pages/home/index`
  今日学习首页，展示当前词书、今日目标、连续学习、正确率、近 7 天趋势和薄弱词入口。
- `pages/books/index`
  词书页，用于查看当前词书、切换词书，并预留“导入与备份”能力入口。
- `pages/review/index`
  复习页，按“到期复习 / 收藏 / 错词”组织待处理队列。
- `pages/me/index`
  “我的”页，展示学习统计和快捷入口。
- `pages/study/index`
  学习会话页，支持展示释义、标记记住/忘记、收藏单词、自动进入下一词。

整体使用路径大致是：

1. 在首页看今日进度和薄弱词
2. 进入学习页做新词或弱词学习
3. 在复习页处理到期词、收藏词、错词
4. 在词书页切换当前学习书
5. 在“我的”页看汇总数据和快捷入口

## 核心状态模型

`miniprogram/utils/types.ts` 和 `miniprogram/utils/state.ts` 是理解项目的关键。

- `AppState`
  保存所有词书、按单词维度的学习进度、每日活动、当前词书和每日目标。
- `WordProgress`
  为每个单词记录学习次数、正确/错误次数、收藏状态、掌握度、最近学习时间、下次复习时间。
- `StudyMode`
  当前有 `study`、`review`、`favorites`、`mistakes` 四种学习模式。

当前学习策略是纯本地规则：

- 默认每日目标 `20`
- 复习掌握阈值 `80`
- 单次学习会话大小 `12`
- 答对时掌握度 `+18`
- 答错时掌握度 `-12`

这套逻辑已经能够驱动首页统计、复习队列和学习卡片三部分联动。

## 内置内容

当前仓库至少包含以下内置词书方向：

- 考研核心词
- CET-6 高频词
- 产品/技术常用英语词
- IT 英语 1700 词

其中 `IT 英语 1700 词` 体量很大，来自单独的数据文件。

## 已知问题与协作风险

- 中文文案存在明显乱码风险
  `app.json`、多个页面的 `wxml`、词书数据里都能看到中文乱码，和你截图里的界面表现一致。后续如果要修 UI 或补文案，编码问题应优先确认。
- `README.md` 还是模板内容
  不要把它当成产品说明或开发现状。
- `ts` 与 `js` 产物共存，且存在不一致
  例如 `seed-wordbooks.ts` 里只有 3 本词书，但 `seed-wordbooks.js` 额外引入了 `IT_ENGLISH_1700_WORD_BOOK`。后续改动词书或学习逻辑时，要先确认微信开发者工具实际运行的是哪份文件，以及是否需要同步维护。
- 目录有一层外包装
  当前工作区根目录不是实际源码根，协作时容易误改到外层配置文件。
- 导入、备份、设置等能力仍是占位
  页面上已经预留入口，但还没有完整实现。

## 推荐协作方式

- 先从 `miniprogram/utils/state.ts` 理解业务规则，再改页面
- 涉及文案或布局问题时，优先检查文件编码和 `wxml` 是否已损坏
- 涉及词书内容时，同时核对 `.ts` 和 `.js` 是否一致
- 涉及新功能时，优先沿用“本地状态驱动”的现有架构，不要默认引入后端依赖

## 一句话总结

这是一个已经做出主流程的微信背单词小程序，优势是学习闭环和本地状态逻辑已经齐备；当前最大的工程风险不是业务缺失，而是文案编码混乱、`ts/js` 双份文件并存以及文档落后于真实实现。
